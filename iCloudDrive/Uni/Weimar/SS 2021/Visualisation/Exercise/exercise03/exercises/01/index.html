Introduction

As you have seen in the lecture, data can come in many different forms: tables, networks, fields and so on. In JavaScript, data is usually modelled as one or more arrays of objects with the same properties but different values.

In this excercise, we will work with artificial, tablular data and focus on basic techniques to create visualisations. using Javascript and d3.js.

Scales

At the very core, implementing a visualization resolves to mapping data items to visual marks and channels. The visualization library d3.js provides the module d3-scale to do just that. It includes scales for different input and output data types and different use cases. For this section, we will focus on the linear scale, that will help you map the max, min, mean and standard deviation of a pool of datasets, that were already loaded for you in the main.js file, to visual elements in the visualization below. To detmine the stats of the input data, you should use the d3-array module.

Join pattern

Now, we want to compare the dataset's stats directly with each other. Therefore, we need another dimension and display all stat visualizations at once one below the other. Here again, d3-scale will provide you with a suitable scale to calculate the vertical position of each stat visualization.

In the task before, we reacted to user interactions by removing and reattatching all SVG elements. This, however, is computationally expensive and causes the browser to slow down quite a bit. Fortunately, d3 has a solution to that problem: The join pattern allows you to specify graphical elements to be created, modified or deleted to match the given input data.

Scatterplot with Time Scale and Animation

Time-based attributes need to be treated specially due to their unusual properties. Fortunately, d3 provides support for mapping time-based data as well in the d3-scale module. Using these, we will have a look at how the datasets, we have created the stats visualizations for, actually look like by creating a scatterplot from the time and y attributes of the datasets.

We will again use the join pattern to reuse the graphical elements on user interaction. To make things a little more interesting, the datapoints will be animated to transition from one state to the other using another d3 module called d3-transition. In our case these transitions are simply a show case to learn how to animate the transition of an element from one state to the other. In cases, where each datapoint represents one specific item and is simply shown in different contexts, animating transitions is neccessary to tell the user what is happening.
