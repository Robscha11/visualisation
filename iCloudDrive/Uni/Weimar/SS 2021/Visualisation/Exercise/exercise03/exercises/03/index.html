Slice and Dice Algorithm

The readability of the parallel sets visualization suffers from the many link overlaps, so another view without overlaps might be helpful. A good choice for an alternative view would be the mosaic plot.

To create a mosaic plot view of the data in the parallel set visualization, the data first has to be transformed with an algorithm called Slice and Dice.

The algorithm builds on the attribute hierarchy already created for the parallel sets in the main.js and is best explained via an image:

The slide is from this slide set about graphs and hierarchies, since the algorithm is also used to create tree maps.

The attribute hierarchy has already been created for you from the attributes and attribute order given in the main.js. It contains a d3-hierarchy, on top of which you can run your algorithm. These notebooks on the functions of a d3-hierarchy and the traversal of a d3-hierarchy will help you accomplishing the task.

A short disclaimer: The coloring of parallel sets and mosaic plot do not match! This is due to the convention in parallel sets of coloring by the first attribute in the hierarchy, which would not add much information to a mosaic plot, since the first split is usually the most easy to see anyway. With mosaic plots, coloring by the second attribute increases the readability of the rows considerably. The additional opacity based on the third attribute within the hierarchy adds further visual clues.
